# Reflected XSS with AngularJS sandbox escape and CSP

Lab ini menggunakan **CSP (Content Security Policy)** dan **AngularJS**.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting (XSS)** yang berhasil **melewati CSP (bypass CSP)**, keluar dari **AngularJS sandbox**, dan menampilkan `alert` `document.cookie`.


## Solusi


1. Buka **exploit server** dan tempelkan kode berikut, lalu ganti **YOUR-LAB-ID** dengan ID lab kamu:

```html 
<script>
location='https://YOUR-LAB-ID.web-security-academy.net/?search=%3Cinput%20id=x%20ng-focus=$event.composedPath()|orderBy:%27(z=alert)(document.cookie)%27%3E#x';
</script>
```

2. Klik **"Store"** dan **"Deliver exploit to victim"**.

---

Eksploit ini menggunakan event **ng-focus** di AngularJS untuk membuat event fokus yang dapat melewati CSP (Content Security Policy).

Eksploit ini juga menggunakan `$event`, yaitu variabel AngularJS yang mereferensikan objek event.

Properti `path` bersifat spesifik untuk Chrome dan berisi array elemen yang memicu event tersebut. Elemen terakhir dalam array tersebut berisi objek `window`.

Biasanya, karakter `|` digunakan sebagai operasi bitwise OR dalam JavaScript, tetapi di AngularJS karakter ini digunakan sebagai indikator operasi filter, dalam kasus ini filter `orderBy`.

Tanda titik dua (`:`) digunakan untuk menandakan argumen yang dikirim ke filter tersebut.

Di dalam argumen itu, alih-alih memanggil fungsi `alert` secara langsung, fungsi tersebut disimpan ke variabel `z`.

Fungsi tersebut hanya akan dipanggil ketika operasi `orderBy` mencapai objek `window` di dalam array `$event.path`.

Dengan cara ini, fungsi dapat dieksekusi dalam scope `window` tanpa referensi eksplisit ke objek `window`, sehingga berhasil melewati pengecekan AngularJS terhadap objek `window`.
