# Reflected XSS into HTML context with most tags and attributes blocked

Lab ini berisi kerentanan **reflected XSS** pada fitur pencarian, tetapi menggunakan **web application firewall (WAF)** untuk melindungi dari vector XSS yang umum.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting** yang melewati WAF dan memanggil fungsi **`print()`**.


> Solusi kamu tidak boleh membutuhkan interaksi dari pengguna. Memanggil `print()` secara manual di browser kamu sendiri tidak akan menyelesaikan lab.


## Solusi

1. Masukkan vector XSS standar, seperti:

```html 
<img src=1 onerror=print()>
```

2. Perhatikan bahwa payload tersebut diblokir. Pada langkah berikutnya, gunakan **Burp Intruder** untuk menguji tag dan attribute mana yang diblokir.

3. Buka browser Burp dan gunakan fitur pencarian pada lab. Kirim request hasil pencarian tersebut ke **Burp Intruder**.

4. Di **Burp Intruder**, ganti nilai search term menjadi:

```html 
<>
```

5. Letakkan kursor di antara tanda kurung sudut lalu klik **Add §** untuk membuat posisi payload. Nilai search term sekarang harus terlihat seperti:

```html 
<§§>
```

6. Buka **[XSS cheat sheet](https://portswigger.net/web-security/cross-site-scripting/cheat-sheet)** dan klik **Copy tags to clipboard**.

7. Pada panel **Payloads**, bagian **Payload configuration**, klik **Paste** untuk memasukkan daftar tag ke daftar payload. Klik **Start attack**.

8. Setelah serangan selesai, periksa hasilnya. Perhatikan bahwa sebagian besar payload menghasilkan response `400`, tetapi payload `body` menghasilkan response `200`.

9. Kembali ke **Burp Intruder** dan ganti search term menjadi:

```html 
<body%20=1>
```

10. Letakkan kursor sebelum karakter `=` lalu klik **Add §** untuk membuat posisi payload. Nilai search term sekarang harus terlihat seperti:

```html id="c9k6xe"
<body%20§§=1>
```

11. Buka **[XSS cheat sheet](https://portswigger.net/web-security/cross-site-scripting/cheat-sheet)** dan klik **Copy events to clipboard**.

12. Pada panel **Payloads**, bagian **Payload configuration**, klik **Clear** untuk menghapus payload sebelumnya. Lalu klik **Paste** untuk memasukkan daftar attribute ke payload list. Klik **Start attack**.

13. Setelah serangan selesai, periksa hasilnya. Perhatikan bahwa sebagian besar payload menghasilkan response `400`, tetapi payload `onresize` menghasilkan response `200`.

14. Pergi ke **exploit server** dan tempel kode berikut, ganti **YOUR-LAB-ID** dengan ID lab kamu:

```html id="7zqk1m"
<iframe src="https://YOUR-LAB-ID.web-security-academy.net/?search=%22%3E%3Cbody%20onresize=print()%3E" onload=this.style.width='100px'>
```

15. Klik **Store and Deliver exploit to victim**.
