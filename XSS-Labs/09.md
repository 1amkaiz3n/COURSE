# Reflected DOM XSS

Lab ini menunjukkan sebuah kerentanan **reflected DOM**. Kerentanan reflected DOM terjadi ketika aplikasi sisi server memproses data dari sebuah request dan memantulkan data tersebut kembali di dalam response. Sebuah script pada halaman kemudian memproses data yang dipantulkan tersebut dengan cara yang tidak aman, hingga akhirnya menulisnya ke sebuah **sink berbahaya**.

Untuk menyelesaikan lab ini, buat sebuah **injection** yang memanggil fungsi **`alert()`**.


## Solusi

1. Buka website target dan gunakan kolom pencarian untuk mencari string percobaan acak, misalnya **"XSS"**.

2. Perhatikan bahwa string tersebut direfleksikan dalam response JSON bernama **`search-results`**.

```json
{"results":[],"searchTerm":"XSS"}
```

1. Buka file **`searchResults.js`** dan perhatikan bahwa response JSON tersebut digunakan dengan pemanggilan fungsi **`eval()`**.

```js
function search(path) {
    var xhr = new XMLHttpRequest();
    xhr.onreadystatechange = function() {
        if (this.readyState == 4 && this.status == 200) {
            eval('var searchResultsObj = ' + this.responseText);
            displaySearchResults(searchResultsObj);
        }
    };
    xhr.open("GET", path + window.location.search);
    xhr.send();
```

2. Dengan mencoba berbagai string pencarian, kamu dapat mengetahui bahwa response JSON melakukan escape terhadap tanda kutip. Namun, karakter **backslash (`\`)** tidak di-escape.

**Di Escape :**
  - `<` -> `&lt;`
  - `>` -> `&gt;`

3. Untuk menyelesaikan lab ini, masukkan search term berikut:

```text
\"-alert(1)}//
```

---

Karena kamu menyisipkan sebuah **backslash** dan website tidak melakukan escape terhadapnya, ketika response JSON mencoba melakukan escape pada karakter tanda kutip pembuka, maka akan menambahkan backslash kedua. Hasil dari **double backslash** tersebut membuat proses escape menjadi batal secara efektif.

Artinya, tanda kutip ganda diproses dalam kondisi tidak ter-escape, sehingga menutup string yang seharusnya berisi search term.

Operator aritmatika (dalam kasus ini operator pengurangan) kemudian digunakan untuk memisahkan ekspresi sebelum fungsi **`alert()`** dipanggil.

Terakhir, tanda kurung kurawal penutup dan dua garis miring digunakan untuk menutup object JSON lebih awal dan mengomentari sisa object yang seharusnya ada.

Hasil akhirnya, response dibuat menjadi seperti berikut:

```json
{"searchTerm":"\\"-alert(1)}//", "results":[]}
```
