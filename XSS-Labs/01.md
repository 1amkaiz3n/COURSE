# DOM XSS di `innerHTML` sink menggunakan source `location.search`

Kerentanan ini terjadi karena `innerHTML` digunakan untuk mengisi konten HTML sebuah elemen `div` dengan data dari `location.search`.

Contoh code di Inspect:

```html
<span id="searchMessage">1amkaiz3n</span>
```

```js id="p8k2qz"
<script>
function doSearchQuery(query) {
    document.getElementById('searchMessage').innerHTML = query;
}

var query = (new URLSearchParams(window.location.search)).get('search');

if (query) {
    doSearchQuery(query);
}
</script>
```

---

## Cara Inject

1. Masukkan string acak alfanumerik ke kotak pencarian.Contoh: `1amkaiz3n`
2. Klik kanan → Inspect element, lalu lihat bahwa input muncul di dalam elemen:
   - `<span id="searchMessage">1amkaiz3n</span>`
3. Uji injeksi dengan keluar dari konteks HTML menggunakan:
   - `">`
4. Lihat di element,ini tidak berhasil keluar dari atribut karean di escape
   - `<span id="searchMessage">"&gt;1amkaiz3n</span>`
5. Lalu lanjutkan payload HTML berbahaya, misalnya:
   - `<img src=1 onerror=alert(1)>`

---

##  Penjelasan
Ini inti yang sering bikin bingung di DOM XSS: **bukan “payload tertentu yang di-escape atau tidak” secara universal, tapi tergantung konteks parsing HTML + filter yang dipakai aplikasi.**

Kita bedah pelan.

---

### 1. Kenapa `">` jadi `&gt;` (di-escape)

Karakter `>` adalah karakter spesial di HTML karena dia bisa:

* menutup tag (`<div> ... >`)
* mengubah struktur DOM

Jadi kalau aplikasi punya filter/encoding, biasanya dia akan:

* mengubah `>` → `&gt;`

Tujuannya:

* mencegah “keluar dari konteks HTML”

Contoh:

```
">Hello
```

jadi:

```
"&gt;Hello
```

Artinya: **browser cuma anggap teks, bukan HTML.**

---

### 2. Kenapa `<img src=1 onerror=alert(1)>` tidak di-escape

Karena ini sudah **bukan “karakter keluar konteks”**, tapi:

> input yang dianggap sebagai HTML valid oleh `innerHTML`

Kalau aplikasi:

```js
innerHTML = userInput;
```

Browser akan:

* parsing string sebagai HTML
* bukan memfilter karakter satu-satu

Jadi:

```html
<img src=1 onerror=alert(1)>
```

langsung dianggap:

* tag HTML baru
* bukan “teks berbahaya yang harus di-escape”

---

### 3. Kenapa kelihatannya “tidak konsisten”

Karena biasanya ada 2 kemungkinan :

#### A. Escape partial (karakter saja)

* `>` → di-escape
* tapi `<` tidak di-block

#### B. No proper sanitization

* input langsung masuk `innerHTML`
* browser yang “kerja sendiri”

---

### 4. Intinya (biar kebayang)

* `">` → dianggap **kontrol karakter**, jadi disanitasi
* `<img ...>` → dianggap **HTML valid**, jadi diparse, bukan di-escape

---

### 5. Analogi gampang

Bayangin:

* `">` = kamu mau “keluar dari pintu”, jadi dijaga satpam
* `<img ...>` = kamu sudah masuk ruangan, tapi kamu bawa “alat baru” yang langsung dipasang di dalam
