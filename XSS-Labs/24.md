# Reflected XSS in a JavaScript URL with some characters blocked

Lab ini menampilkan (reflect) input kamu di dalam sebuah **JavaScript URL**, tetapi tidak semuanya seperti yang terlihat.

Awalnya ini terlihat seperti tantangan yang sederhana; namun, aplikasi ini memblokir beberapa karakter sebagai upaya untuk mencegah serangan XSS.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting (XSS)** yang memanggil fungsi `alert`, dengan string `1337` yang terdapat di dalam pesan `alert` tersebut.


## Solusi

Kunjungi URL berikut, dengan mengganti **YOUR-LAB-ID** dengan ID lab kamu:

```text
https://YOUR-LAB-ID.web-security-academy.net/post?postId=5&%27},x=x=%3E{throw/**/onerror=alert,1337},toString=x,window%2b%27%27,{x:%27
```

Lab akan berhasil diselesaikan, tetapi alert hanya akan dipanggil jika kamu mengklik **"Back to blog"** di bagian bawah halaman.

---

Eksploit ini menggunakan **exception handling** untuk memanggil fungsi alert dengan argumen tertentu.

Statement `throw` digunakan, dipisahkan dengan komentar kosong agar bisa melewati pembatasan larangan spasi.

Fungsi `alert` ditetapkan ke handler exception `onerror`.

Karena `throw` adalah sebuah statement, ia tidak bisa digunakan sebagai expression. Oleh karena itu, digunakan **arrow function** untuk membuat sebuah blok agar statement `throw` bisa dipakai.

Kemudian fungsi ini dipanggil dengan cara menetapkannya ke properti `toString` milik `window`, lalu memicunya dengan memaksa konversi `window` menjadi string.
