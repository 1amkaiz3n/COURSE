# XSS

## Reflected XSS

Reflected XSS terjadi ketika input pengguna langsung dikembalikan oleh aplikasi web dalam pesan kesalahan, hasil pencarian, atau respons lain yang mencakup sebagian atau seluruh input yang diberikan oleh pengguna sebagai bagian dari permintaan, tanpa data tersebut diamankan untuk ditampilkan di browser, dan tanpa menyimpan data yang diberikan pengguna secara permanen. 

Contohnya :- `https://xss.challenge.training.hacq.me/challenges/baby01.php?payload=%22%3E%3Cimg/src/onerror=alert(0)%3E`

Di sini `">` tidak dienkode oleh aplikasi web yang menyebabkan XSS

## DOM BASED XSS

XSS (Cross-Site Scripting) berbasis DOM adalah jenis serangan XSS yang mengeksploitasi kerentanan dalam skrip sisi klien dan Document Object Model (DOM) dari halaman web untuk mengeksekusi kode berbahaya di browser korban. 

Jadi, sederhananya, ada kode yang rentan yang dapat digunakan oleh penyerang untuk memanipulasi DOM agar menjalankan skrip kita. 

MIsalknya :

```html
<!DOCTYPE html>
<html>
<head>
<title>XSS berbasis DOM/title>
<script>
var name = window.location.hash.substring(1);
document.write('<h1>Halo, ' + name + '</h1>');
</script>
</head>
<body>
</body>
</html> 
```

Tag `<script>` digunakan untuk menyertakan kode JavaScript dalam dokumen. Dalam hal ini, skrip mengambil nilai dari `hash` Bagian dari URL tersebut digunakan untuk membuat pesan sapaan. Pesan tersebut kemudian ditulis ke dokumen HTML menggunakan `document.write`. 



jadi `http://example.com/# <script>alert()</script>`

akan digunakan untuk menjalankan skrip

atau seperti ada parameter di URL seperti `?url=https://test.com`

Tag `<script>` mencerminkan hal tersebut

variabel `x = 'https://test.com/'`
