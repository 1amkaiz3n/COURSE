#  Exploiting an API endpoint using documentation

Untuk menyelesaikan lab ini, temukan dokumentasi API yang terekspos dan hapus akun `carlos`. Kamu bisa login ke akunmu sendiri menggunakan kredensial berikut: `wiener:peter`.

**Pengetahuan yang dibutuhkan**

Untuk menyelesaikan lab ini, kamu perlu memahami:

* Apa itu dokumentasi API.
* Bagaimana dokumentasi API bisa berguna bagi penyerang.
* Cara menemukan dokumentasi API.

Poin-poin ini dibahas dalam topik **[API Testing Academy](https://portswigger.net/web-security/api-testing)**.

Singkatnya: sistem ini punya “bocoran manual rahasia”, dan kamu diminta menemukannya lalu memanfaatkannya. Karena tentu saja, kenapa bikin API aman kalau bisa ditinggal begitu saja di internet, kan.


## Solusi

1. Di browser, login ke aplikasi menggunakan kredensial **wiener:peter**, lalu perbarui alamat email kamu.

2. Di **Proxy > HTTP history**, klik kanan request **PATCH /api/user/wiener**, lalu pilih **Send to Repeater**.

3. Buka tab **Repeater**. Kirim request **PATCH /api/user/wiener** tersebut. Perhatikan bahwa ini mengambil kredensial untuk user **wiener**.

```json
{"username":"wiener","email":"testing@gmail.com"}
```

4. Hapus **/wiener** dari path request, sehingga endpoint menjadi **/api/user**, lalu kirim request. Perhatikan bahwa ini menghasilkan error karena tidak ada identifier user.

```json
{"error":"Malformed URL: expecting an identifier"}
```

5. Hapus juga **/user** dari path request, sehingga endpoint menjadi **/api**, lalu kirim request. Perhatikan bahwa ini mengambil dokumentasi API.

6. Klik kanan pada response, lalu pilih **Show response in browser**. Salin URL yang muncul.

7. Tempel URL tersebut ke browser Burp untuk mengakses dokumentasi. Perhatikan bahwa dokumentasinya bersifat interaktif.

8. Untuk menghapus **carlos** dan menyelesaikan lab, klik baris **DELETE**, masukkan **carlos**, lalu klik **Send request**.

Ya, intinya kamu cuma “mengupas” endpoint sampai API-nya menyerah dan ngasih dokumentasi sendiri.
