# Exploiting a mass assignment vulnerability

Untuk menyelesaikan lab ini, temukan dan eksploitasi kerentanan **mass assignment** untuk membeli jaket kulit **Lightweight l33t Leather Jacket**. Kamu bisa login ke akunmu sendiri menggunakan kredensial berikut: `wiener:peter`.


## Solusi 

1. Di browser Burp, login ke aplikasi menggunakan kredensial `wiener:peter`.

2. Klik produk **Lightweight "l33t" Leather Jacket** lalu tambahkan ke keranjang.

3. Buka keranjang kamu lalu klik **Place order**. Perhatikan bahwa kamu tidak memiliki cukup credit untuk pembelian tersebut.

4. Di **Proxy > HTTP history**, perhatikan request API `GET` dan `POST` untuk `/api/checkout`.

5. Perhatikan bahwa response dari request `GET` berisi struktur JSON yang sama dengan request `POST`. Di dalam response GET, terdapat parameter `chosen_discount`, yang tidak ada di request POST.

```json
{"chosen_discount":{"percentage":0},"chosen_products":[{"product_id":"1","name":"Lightweight \"l33t\" Leather Jacket","quantity":1,"item_price":133700}]}
```

6. Klik kanan request `POST /api/checkout` lalu pilih **Send to Repeater**.

7. Di Repeater, tambahkan parameter `chosen_discount` ke dalam request. JSON-nya harus seperti berikut:

```json
{
    "chosen_discount": {
        "percentage": 0
    },
    "chosen_products": [
        {
            "product_id": "1",
            "quantity": 1
        }
    ]
}
```

8. Kirim request tersebut. Perhatikan bahwa penambahan parameter `chosen_discount` tidak menghasilkan error.

9. Ubah nilai `chosen_discount` menjadi string `"x"`, lalu kirim request. Perhatikan muncul error karena nilai parameter bukan angka. Ini mengindikasikan input pengguna sedang diproses oleh server.

10. Ubah nilai `percentage` pada `chosen_discount` menjadi `100`, lalu kirim request untuk menyelesaikan lab.
