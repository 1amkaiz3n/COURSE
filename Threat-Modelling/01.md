# Threat Modelling

Threat modeling itu adalah merupakan sebuah struktur yang direpresentasikan dalam berbagai macam komponen, yang dimana komponen-komponen ini nantinya akan dirancang sedemikian rupa sehingga bisa memahami bagaimana ancaman itu bekerja di dalam sistem atau sebuah perusahaan.

Tujuannya agar kita bisa mengidentifikasi ancaman-ancaman tersebut, untuk menangani ancaman itu sendiri, memahami ancaman itu sendiri seperti apa, cara kerjanya bagaimana, masuknya lewat mana, mengancam apa, dan cara mitigasinya bagaimana.

Threat modelling ini akan berbentuk diagram, diagram ini terdiri dari berbagai komponen-komponen.
Secara umum, komponen threat modeling ada 4:

1. Subject

   1. Process
   2. Store
2. Flow
3. Aktor
4. Boundary

---

1. Aktor
   = pihak yang berinteraksi dengan sistem

Contoh:

* webmaster
* user
* admin
* attacker
* third-party service
* payment provider

2. Flow
   = aliran data atau komunikasi antar komponen

Contoh:

* login request
* API request
* token
* cookie
* file upload
* webhook

Biasanya digambarkan dengan panah.

3. Subject
   = komponen atau object di dalam sistem

* Process
  = komponen yang memproses data

  Contoh:

  * frontend
  * API
  * auth service
  * queue worker

* Store
  = tempat penyimpanan data

  Contoh:

  * database
  * storage
  * Redis
  * session store

Biasanya digambarkan dengan kotak.

4. Boundary
   = batas trust atau perubahan privilege di dalam sistem

Contoh:

* guest ↔ login
* public ↔ internal
* user ↔ admin
* internet ↔ internal network

Biasanya digambarkan dengan garis pemisah.

---

Threat modeling dapat diterapkan di berbagai area seperti:

* software
* aplikasi web
* sistem
* jaringan
* distributed system
* cloud
* IoT
* mobile application
* infrastructure
* microservice architecture
